# Ralph 项目分析

## 一、架构设计

### 1.1 整体架构

Ralph 是一个**自主 AI 代理循环系统**，采用**迭代式执行架构**，核心设计理念是"每次迭代都是全新的上下文"。

```
┌─────────────────────────────────────────────────────────────┐
│                      Ralph 核心循环                           │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐   │
│  │  读取 PRD    │───▶│  实现 Story  │───▶│  质量检查    │   │
│  │  (prd.json)  │    │  (代码实现)  │    │  (typecheck) │   │
│  └──────────────┘    └──────────────┘    └──────────────┘   │
│         │                    │                    │         │
│         ▼                    ▼                    ▼         │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐   │
│  │  选择 Story   │    │  提交代码    │    │  更新状态    │   │
│  │  (优先级)    │    │  (git commit)│    │  (prd.json)  │   │
│  └──────────────┘    └──────────────┘    └──────────────┘   │
│         │                    │                    │         │
│         └────────────────────┴────────────────────┘         │
│                              │                               │
│                              ▼                               │
│                    ┌──────────────┐                         │
│                    │  记录学习    │                         │
│                    │  (progress.txt)│                        │
│                    └──────────────┘                         │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| 核心脚本 | Bash | `ralph.sh` - 主循环控制器 |
| AI 工具 | Amp / Claude Code | 支持两种 AI 编码工具 |
| 可视化 | React Flow | 交互式流程图展示 |
| 数据格式 | JSON | `prd.json` - 用户故事状态管理 |
| 日志 | 纯文本 | `progress.txt` - 学习记录 |

### 1.3 核心文件结构

```
ralph/
├── ralph.sh              # 主循环脚本
├── prompt.md             # Amp 提示词模板
├── CLAUDE.md             # Claude Code 提示词模板
├── prd.json.example      # PRD JSON 格式示例
├── AGENTS.md             # 项目级代理指令
├── skills/               # AI 技能模块
│   ├── prd/              # PRD 生成技能
│   └── ralph/            # PRD 转换技能
├── flowchart/            # 可视化流程图
│   └── src/App.tsx       # React Flow 实现
└── .claude-plugin/       # Claude Code 插件配置
```

## 二、功能定位

### 2.1 核心价值

Ralph 解决的核心问题是：**如何让 AI 编码工具自主完成复杂的功能开发任务**。

传统 AI 编码工具的局限：
- 上下文窗口限制
- 无法处理多步骤任务
- 缺乏持续学习和记忆

Ralph 的解决方案：
- **迭代式执行**：将大任务拆分为小故事
- **状态持久化**：通过 git、prd.json、progress.txt 保持记忆
- **自主循环**：自动重复执行直到所有任务完成

### 2.2 目标用户

- **个人开发者**：快速实现功能原型
- **小团队**：自动化重复性开发工作
- **AI 实验者**：探索 AI 自主编程的边界

### 2.3 使用场景

```
1. 功能开发
   └── 将 PRD 转化为可执行的代码

2. 代码重构
   └── 按步骤完成重构任务

3. 测试补充
   └── 为现有代码添加测试用例

4. 文档完善
   └── 生成和更新技术文档
```

## 三、功能核心亮点

### 3.1 自主迭代循环

**亮点**：每次迭代都是全新的 AI 实例，但通过外部状态保持连续性。

```bash
# 核心循环逻辑 (ralph.sh)
for i in $(seq 1 $MAX_ITERATIONS); do
  # 1. 启动新的 AI 实例（干净上下文）
  OUTPUT=$(amp --dangerously-allow-all < prompt.md)
  
  # 2. 检查完成信号
  if echo "$OUTPUT" | grep -q "<promise>COMPLETE</promise>"; then
    exit 0
  fi
  
  # 3. 继续下一轮
done
```

**优势**：
- 避免上下文污染
- 每次迭代都有清晰的起点
- 失败不会导致整个任务崩溃

### 3.2 故事驱动开发

**亮点**：将 PRD 拆分为可独立验证的用户故事。

```json
{
  "userStories": [
    {
      "id": "US-001",
      "title": "Add priority field to database",
      "acceptanceCriteria": [
        "Add priority column to tasks table",
        "Generate and run migration successfully",
        "Typecheck passes"
      ],
      "passes": false,  // 状态标记
      "priority": 1     // 执行顺序
    }
  ]
}
```

**优势**：
- 每个故事可独立完成
- 清晰的验收标准
- 可追踪的进度

### 3.3 知识积累机制

**亮点**：通过 `progress.txt` 和 `AGENTS.md` 实现跨迭代学习。

```
# progress.txt 结构
## Codebase Patterns
- 通用模式总结

## [Date/Time] - [Story ID]
- 实现内容
- 文件变更
- **Learnings for future iterations:**
  - 发现的模式
  - 遇到的坑
  - 有用的上下文
```

**优势**：
- 避免重复错误
- 积累项目知识
- 提升后续迭代效率

### 3.4 多工具支持

**亮点**：同时支持 Amp 和 Claude Code 两种 AI 工具。

```bash
# 工具选择
./ralph.sh --tool amp      # 使用 Amp
./ralph.sh --tool claude   # 使用 Claude Code
```

**优势**：
- 工具灵活性
- 便于对比效果
- 降低工具依赖风险

### 3.5 可视化流程

**亮点**：React Flow 交互式流程图，直观展示 Ralph 工作原理。

**功能**：
- 步骤动画展示
- 节点拖拽调整
- 连接线动态显示
- 代码示例标注

## 四、设计核心亮点

### 4.1 无状态迭代设计

**设计哲学**：每次迭代都是全新的开始，状态完全外部化。

```
┌─────────────────────────────────────────────────────────┐
│                    迭代边界                              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   AI 实例 (无记忆)                                        │
│   ┌─────────────────────────────────────────────────┐   │
│   │  读取外部状态 → 执行任务 → 更新外部状态          │   │
│   │  • prd.json (任务状态)                          │   │
│   │  • progress.txt (学习记录)                      │   │
│   │  • git history (代码历史)                       │   │
│   └─────────────────────────────────────────────────┘   │
│                                                         │
│   外部状态 (持久化)                                      │
│   • 不受 AI 实例生命周期影响                               │
│   • 可被人类直接编辑                                     │
│   • 支持版本控制                                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**设计优势**：
- **容错性**：单个迭代失败不影响整体
- **可调试性**：可随时检查中间状态
- **可干预性**：人类可随时修改状态文件

### 4.2 技能系统架构

**设计亮点**：通过技能模块实现功能扩展。

```
skills/
├── prd/
│   └── SKILL.md      # PRD 生成技能
│       ├── 触发条件
│       ├── 执行流程
│       └── 输出格式
└── ralph/
    └── SKILL.md      # PRD 转换技能
        ├── 转换规则
        ├── 故事拆分策略
        └── 优先级排序
```

**设计优势**：
- **模块化**：技能独立开发和测试
- **可复用**：技能可在不同项目间共享
- **可扩展**：易于添加新技能

### 4.3 反馈循环设计

**设计亮点**：多层反馈机制确保质量。

```
┌─────────────────────────────────────────────────────────┐
│                    反馈循环                              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   1. 代码质量反馈                                        │
│      └── typecheck / lint / test                        │
│                                                         │
│   2. 功能验证反馈                                        │
│      └── dev-browser skill (UI 验证)                     │
│                                                         │
│   3. 知识反馈                                            │
│      └── progress.txt 记录学习                          │
│                                                         │
│   4. 状态反馈                                            │
│      └── prd.json 更新 passes 状态                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**设计优势**：
- **质量保证**：多层验证确保代码正确性
- **持续改进**：学习记录提升后续迭代质量
- **进度透明**：状态文件清晰展示进展

### 4.4 故事拆分策略

**设计亮点**：严格的"单迭代可完成"原则。

```
故事大小判断标准:
├── ✅ 合适大小
│   ├── 添加数据库字段和迁移
│   ├── 添加 UI 组件到现有页面
│   ├── 更新服务器操作逻辑
│   └── 添加过滤器下拉框
│
└── ❌ 过大 (需拆分)
    ├── "构建整个仪表板"
    ├── "添加认证系统"
    └── "重构 API"
```

**设计优势**：
- **可控性**：每个故事可在有限上下文中完成
- **可验证性**：小变更易于测试和验证
- **可回滚性**：失败影响范围小

### 4.5 归档机制

**设计亮点**：自动归档不同功能的执行记录。

```bash
# ralph.sh 中的归档逻辑
if [ "$CURRENT_BRANCH" != "$LAST_BRANCH" ]; then
  # 检测到新功能，归档旧记录
  ARCHIVE_FOLDER="$ARCHIVE_DIR/$DATE-$FOLDER_NAME"
  cp prd.json "$ARCHIVE_FOLDER/"
  cp progress.txt "$ARCHIVE_FOLDER/"
  # 重置进度文件
  echo "# Ralph Progress Log" > progress.txt
fi
```

**设计优势**：
- **历史保留**：不丢失之前的执行记录
- **状态隔离**：不同功能互不干扰
- **可追溯性**：可随时查看历史执行结果

---

## 五、总结

Ralph 是一个精心设计的 AI 自主编程系统，其核心创新在于：

1. **架构层面**：无状态迭代 + 外部状态持久化
2. **功能层面**：故事驱动 + 自主循环 + 知识积累
3. **设计层面**：模块化技能 + 多层反馈 + 严格的故事拆分

这套设计使得 AI 能够可靠地完成复杂的编程任务，同时保持了系统的可调试性和可干预性。
